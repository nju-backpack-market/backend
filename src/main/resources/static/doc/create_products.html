POST/PUT /products

Create/update products

Format:

 
x
[
  {
    "pid": [pid],
    "name": [name],
    "price": [price],
    "description": [description],
  }...,
]

Example

 
xxxxxxxxxx
[
  {
    "pid": 1,
    "name": "MockProduct",
    "price": 10000,
    "description": "MOCK"
  },
  {
    "pid": 2,
    "name": "MockProduct",
    "price": 10005,
    "description": "MOCK"
  }
]

Note:

  • pid is ignored when creating, and used as identifier when updating.

  • when updating, some fields of pass-in parameter can be absent. Those fields keep unchanged.

  • returning list of products with full information, which are successfully created/updated. This means,

    • generated id is contained
    • creating always fails/succeed all together
    • and updating can partially succeed, only successful ones are returned.
