PUT /orders

update order's info

Format

 
x
{
  "id": [id],
  "status": [status],
  "delivery_info": {
    "name": [name],
    "phone_number": [phone_number],
    "email": [email],
    "country": [country],
    "city": [city],
    "address_line1": [address_line1],
    "address_line2": [address_line2],
    "postal_code": [postal_code]
  }
}

​

Example

Example:

 
x
{
  "id": 5,
  "status": CREATE,
  "delivery_info": {
    "name": "zhangsan",
    "phone_number": "1234567890",
    "email": "foo@bar.com",
    "country": "China",
    "city": "南京",
    "address_line1": "栖霞区仙林大道168号",
    "address_line2": "南大和园99栋909室",
    "postal_code": "210023"
  }
}

NOTE

  • 格式的约定和创建/修改商品差不多。即id必选其他可选，缺少的字段不会修改。返回修改后的列表。

  • 这个接口不会返回表示有更新失败的状态值。即使全部失败也是返回200和空列表。

  • 只允许修改status和delivery_info，其他的不允许修改。

  • 只要修改状态的话推荐使用update_order_status的接口。

  • 状态不能随便修改，比如CREATE状态只允许变成PAYING和CLOSED。此处可能变更（主要是异常处理流程）正常的转换如下:

    CREATE -> PAYING | CLOSED // 已创建

    PAYING -> CREATE | STOCK_OUT // 正在支付

    STOCK_OUT -> DELIVERING // 正在出库

    DELIVERING -> SIGNED // 正在配送

    SIGNED -> COMMENTED // 已签收

    COMMENTED -> COMPLETED // 已评论

    // 最后一个是已完成，没有合法的转换

  • 权限接口, 这个接口用来给后台强行改信息用
