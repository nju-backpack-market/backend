PUT /orders

update order's info

Format

 
{
  "id": [id],
  "status": [status],
  "delivery_info": {
    "name": [name],
    "phone_number": [phone_number],
    "email": [email],
    "address": [address]
  }
}

​

Example

Example:

 
xxxxxxxxxx
{
  "id": 1,
  "status": CREATE,
  "delivery_info": {
    "name": "zhangsan",
    "phone_number": "1234567890",
    "email": "foo@bar.com",
    "address": "foobar"
  }
}

NOTE

  • 格式的约定和创建/修改商品差不多。即id必选其他可选，缺少的字段不会修改。返回修改后的列表。
  • 这个接口不会返回表示有更新失败的状态值。即使全部失败也是返回200+空列表。
  • 只允许修改status和delivery_info，其他的不允许修改。
  • 只要修改状态的话推荐使用update_order_status的接口。
