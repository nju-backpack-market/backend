GET /orders

关于各种价格：

所有价格>=0.0

origin 开头的表示此订单创建时的原价

actual开头的表示此订单创建时的实际价格 ，实际价格可能是经过折扣过后的。

unit_price表示单个商品的价格

subtotal_price表示单种商品（可能是多个）的价格

total_price表示这个订单的总价

origin_subtotal_price = origin_unit_price * count

origin_total_price = sum(origin_subtotal_price)

但是actual_unit_price, actual_subtotal_price, actual_total_price 之间不一定有关系。

actual_total_price就是最终的支付价格。

  • GET /orders?full={full} 获取所有订单

    参数：full 可选，默认false。false表示返回的每一个order中不包含关联的shopping items。请注意此时所有origin的price都是0，actual的price是正确的。true表示包含关联的shopping item，此时origin的price是对的。
